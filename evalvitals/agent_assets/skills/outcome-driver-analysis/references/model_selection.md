# Model selection reference

Dense decision tables supporting `SKILL.md` steps 3-6. Read the relevant table when
you reach that step; don't load this whole file into a report.

## Step 3 -- univariate test vs. outcome, by variable type

| Variable type | Assumptions met | Primary test | Assumptions violated / robust fallback |
|---|---|---|---|
| Categorical (vs. binary outcome) | all expected cell counts >= 5 | Chi-square test of independence | Fisher's exact test (small/sparse cells); Cochran-Mantel-Haenszel test if stratifying by a third categorical variable |
| Continuous (vs. binary outcome) | approx. normal within each group, similar variances | Two-sample t-test (Student's) | Welch's t-test (unequal variances); Mann-Whitney U / Wilcoxon rank-sum (non-normal, robust default); permutation test (small n, no distributional assumption) |
| Ordinal (vs. binary outcome) | -- | Mann-Whitney U / Wilcoxon rank-sum | Cumulative-link / ordinal logistic model if treating ordinal as a predictor in a larger model |

Effect sizes to report alongside each test: Cramer's V or odds ratio (categorical),
Cohen's d or rank-biserial correlation (continuous/ordinal).

R: `chisq.test()`, `fisher.test()`, `t.test()`, `wilcox.test()`.
Python: `scipy.stats.chi2_contingency`, `scipy.stats.fisher_exact`, `scipy.stats.ttest_ind`,
`scipy.stats.mannwhitneyu`.

## Step 4 -- marginal screening

| Screening approach | When to use | R | Python |
|---|---|---|---|
| Univariate logistic regression per variable, rank by p-value | Default screening method | `glm(outcome ~ x, family = binomial)` | `statsmodels.api.Logit` |
| AIC improvement over null model | Comparing variables on a consistent scale, useful when p-values are similar across many variables | `AIC(model_null) - AIC(model_x)` | `model_x.aic` vs. null model `aic` |
| Reuse step-3 bivariate test p-values | Quick pass when there are few candidate variables | -- | -- |

Screening narrows the candidate list; it does not replace fitting the full
multivariable model in step 5. Keep a variable in the candidate set even if its
marginal signal is weak when subject-matter reasoning or the step-3 conditioning
check suggests a real, confounded effect.

## Step 5 -- outcome type and clustering -> model family

| Outcome type | Observations independent? | Model | R | Python |
|---|---|---|---|---|
| Binary (success/fail) | Yes | Logistic regression (binomial GLM) | `glm(y ~ ., family = binomial)` | `statsmodels.api.Logit` / `sklearn.linear_model.LogisticRegression` (use `statsmodels` for inference/p-values) |
| Binary (success/fail) | No -- clustered/repeated (e.g. multiple examples per model/prompt/subject) | Mixed-effects logistic regression (GLMM) | `lme4::glmer(y ~ . + (1\|cluster), family = binomial)`, or `glmmTMB` | `statsmodels.genmod.bayes_mixed_glm.BinomialBayesMixedGLM`, or `pymer4` (R-backed) |
| Continuous | Yes | Linear regression | `lm(y ~ .)` | `statsmodels.api.OLS` |
| Continuous | No -- clustered/repeated | Linear mixed-effects model (LMM) | `lme4::lmer(y ~ . + (1\|cluster))` | `statsmodels.regression.mixed_linear_model.MixedLM` |
| Count | Yes | Poisson or negative binomial regression (check for overdispersion) | `glm(y ~ ., family = poisson)`, `MASS::glm.nb()` | `statsmodels.discrete.discrete_model.Poisson`, `.NegativeBinomial` |
| Count | No -- clustered/repeated | Mixed-effects Poisson/negative binomial | `glmmTMB(y ~ . + (1\|cluster), family = nbinom2)` | `statsmodels` mixed count models are limited -- R is the more mature option here |

Why not linear regression for a binary outcome: predictions can fall outside [0, 1],
and the constant-variance / normal-error assumptions behind linear regression's
inference (t-tests, F-tests, prediction intervals) don't hold for a 0/1 target.
Logistic regression models the log-odds of success directly and its errors match a
Bernoulli outcome.

When to add a random effect (mixed-effects model): observations share a grouping
structure that could induce correlated errors -- e.g. several examples generated by
the same underlying model, drawn from the same prompt template, or measured on the
same subject. Ignoring this structure understates standard errors and can produce
false-positive findings. A rule of thumb: if you can name the grouping variable and
each group contributes more than one row, consider a random intercept for it; add a
random slope only if there's reason to think the effect of a predictor itself varies
by group and there's enough data per group to estimate it.

## Step 6 -- goodness-of-fit and diagnostics, by model

| Check | Applies to | R | Python |
|---|---|---|---|
| Hosmer-Lemeshow test | Logistic regression (few continuous predictors) | `ResourceSelection::hoslem.test()` | `statsmodels` has no built-in; implement via binned deciles of predicted probability |
| Deviance / Pearson residuals | Any GLM | `residuals(model, type = "deviance")` | `model.resid_deviance`, `model.resid_pearson` |
| Binned residual plot | Logistic regression / GLMM | `arm::binnedplot()` | plot residuals vs. fitted, binned manually |
| VIF (multicollinearity) | Any regression with >1 predictor | `car::vif(model)` | `statsmodels.stats.outliers_influence.variance_inflation_factor` |
| Influence diagnostics | Any GLM | `influence.measures(model)` | `model.get_influence()` |
| ROC / AUC | Binary outcome model | `pROC::roc()` | `sklearn.metrics.roc_curve`, `roc_auc_score` |
| Calibration plot | Binary outcome model | `rms::val.prob()` | manual: bin predicted probability, plot observed vs. predicted rate |
| Random-effect variance / ICC | GLMM/LMM | `performance::icc(model)` | `pymer4` wraps R; native Python support is limited |
| Convergence check | GLMM/LMM | inspect `model@optinfo$conv` | inspect optimizer convergence flag on the fit result |
