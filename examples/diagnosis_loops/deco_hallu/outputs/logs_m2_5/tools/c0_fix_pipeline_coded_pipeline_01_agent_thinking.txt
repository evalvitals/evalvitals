**Strategy — require localized spatial corroboration before answering "Yes":**

1. **Trust "No" answers immediately.** Since the failure is *over-detection* from a "Yes" bias, a baseline "No" is almost never a false negative — so I accept it without spending more calls (budget-efficient).

2. **Challenge every "Yes" with context-stripped crops.** The language prior is a *scene co-occurrence* effect ("cars usually go with this kind of scene → Yes"). Cropping into overlapping quadrants/center removes the global scene context that feeds the prior, while *magnifying* any genuinely-present (often small) object. A real object survives in at least one crop; a hallucinated one collapses to "No" once its scene cues are gone.

3. **Require ≥2 corroborating crops** (with a `zoom_center` tiebreaker) to confirm "Yes". This raises the evidence threshold specifically against the Yes-bias.

Notes on why this respects the hypotheses:
- I deliberately **don't rely on `model_attend`** for grounding — the attention sink (fixed patches ~58/59/69/80) is content-independent, so cropping "where it looks" would be meaningless. Spatial corroboration via independent crops sidesteps the sink entirely.
- Prompt reformatting is avoided since the prior is immune to it; I attack it through the *image/context* channel instead.
- Budget: typically 1 call for No-cases, up to ~6 for Yes-cases.

Every case emits a clear Yes/No answering the original question.
